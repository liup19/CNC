# CNC 加工定位精度分析 — 刀具移动不一定准确

> 最后更新：2026-05-16

---

## 核心事实

刀具说"我去 X=50"，实际可能到了 X=50.1 或 X=49.95。G-code 是指令，不是保证。从指令到实际运动，中间有多层误差。

```
误差传递链：

  G-code: G0 X50
      ↓
  控制器计算脉冲数
      ↓ 误差 ①：计算精度
  发出电脉冲给电机
      ↓ 误差 ②：丢步
  电机旋转
      ↓ 误差 ③：反向间隙
  丝杠把旋转变直线
      ↓ 误差 ④：丝杠精度
  刀具移动到位置
      ↓ 误差 ⑤：切削力让刀具偏转
  实际位置 ≠ X=50
```

---

## 误差 ① — 反向间隙（Backlash）

### 丝杠、螺母、电机的物理位置

以 Y 轴为例（X、Z 轴结构一样）：

```
正视侧视图（看机器的侧面）：

                  Z 轴电机
                    ┌──┐
                    │M3│
                    └──┤
                       │ 主轴
                    ┌──┤
                    │刀│
                    └──┘

  ┌──────────────────────────────────────┐ ← 横梁（X 轴滑轨）
  │                                      │
  │          X 轴皮带/丝杠               │
  │  ═══════════════════════════         │
  │                                      │
  └──────────────────────────────────────┘
            ↑
            这个横梁整体沿 Y 轴前后移动
            ↓

┌───┐                               ┌───┐
│M2 │                               │M1 │   ← 两个 Y 轴电机
│电机│                               │电机│      固定在机架上
└─┬─┘                               └─┬─┘
  │                                   │
  ╠═══  Y轴丝杠（或皮带）  ═══════════╣
  ║                                   ║
  ║    丝杠沿 Y 方向贯穿整个行程       ║
  ║                                   ║
  ╠═╗                             ╔═╣
  ║ ║← 螺母（固定在横梁底部）       ║ ║← 螺母
  ║ ╚═════════════════════════════╝ ║
  ║                                 ║
──╫─────────────────────────────────╫── ← 机架（固定不动）
  ║                                 ║
```

```
拆开看 Y 轴单个传动链：

  固定端（机架）                       固定端（机架）
  ┌───┐                                ┌───┐
  │电机│                                │轴承│
  │ M │                                │   │
  └─┬─┘                                └─┬─┘
    │                                      │
    │  联轴器                               │
    │  ╔══╗                                │
    └──╢  ╠──────── 丝杠 ═══════════════───┘
       ╚══╝
       ║
    ═══╬════════════════════════════════════
       ║
       ╠═╗  螺母（固定在横梁上，随横梁移动）
       ║ ║← ┌──────────────────────────┐
       ║ ║  │     横梁（移动部件）      │
       ╚═╝  └──────────────────────────┘

  电机转动 → 联轴器带动丝杠转动
  丝杠转动 → 螺母沿丝杠前后移动
  螺母固定在横梁上 → 横梁跟着移动
```

```
各部件的角色：

  电机（固定在机架上）：
    提供动力，输出旋转运动
    位置：轴的端部，固定不动

  丝杠（横跨整个行程）：
    把旋转运动变成直线运动的"轨道"
    两端由轴承支撑
    位置：沿轴方向贯穿，自身转但不移动

  螺母（固定在移动部件上）：
    拧在丝杠上的"滑块"
    丝杠转 → 螺母被迫沿丝杠滑动
    位置：卡在移动部件（横梁/滑块）上，随移动部件一起走
```

```
反向间隙在哪 — 在丝杠和螺母的螺纹之间：

  放大看螺母和丝杠的接触：

    丝杠螺纹    间隙    螺母螺纹
    ═══╪       ←→       ╪═══
    ═══╪                 ╪═══
    ═══╪       ←→       ╪═══
         这个间隙就是反向间隙

  丝杠正转 → 螺纹推螺母的 A 面 → 前进
  丝杠反转 → 丝杠先转过间隙 → 碰到螺母的 B 面 → 才开始后退
  穿过间隙的过程中 → 电机转了但螺母没动 → 丢位置
```

```
三种传动的区别：

  丝杠传动（3018 / 桌面铣床）：
    电机 → 联轴器 → 丝杠（旋转）→ 螺母（直线移动）
    反向间隙：0.005-0.15mm（取决于丝杠精度）

  皮带传动（Shapeoko / X-Carve）：
    电机 → 皮带轮 → 皮带（直线移动）→ 夹在移动部件上
    皮带绕两个皮带轮之间
    ┌───┐                          ┌───┐
    │电机│ → ╔══════════════════╗ → │轮 │
    │皮带轮│  ║皮带（带动横梁）  ║   │  │
    └───┘  ← ╚══════════════════╝ ← └───┘
    反向间隙：0.03-0.15mm（皮带弹性更大）

  滚珠丝杠（桌面铣床/高端）：
    和普通丝杠一样，但丝杠和螺母之间有滚珠
    滚动摩擦代替滑动摩擦 → 精度更高、磨损更小
    反向间隙：0.005-0.02mm
```

### 为什么机器不知道

机器是**数自己发了多少个脉冲**，不是量自己实际走了多远。

```
开环步进电机（消费级）的工作方式：

  控制器发出 1000 个脉冲
  → 电机应该转 1000 步
  → 每步移动 0.01mm
  → 控制器认为：位置 = 起点 + 1000 × 0.01 = 移动了 10mm

  控制器的"位置"就是一个计数器：
    正转 → 计数器 +1
    反转 → 计数器 -1
    计数器 × 每步距离 = "我知道我在哪"

  但它从来没有实际量过自己走了多远
  它只是"假设"每个脉冲都变成了真实的移动
```

```
反向间隙时发生了什么：

  正向移动，螺母紧贴丝杠右侧：
    脉冲 → 电机转 → 丝杠转 → 螺母移动 ✓ 每步都有效

    电机  ═══→  丝杠  ═══→  螺母
                               ║ 紧贴
                          ────────

  反向时，螺母要先走过间隙：

    电机收到 50 个反向脉冲 → 电机确实反转了 50 步
    丝杠也确实反转了
    但螺母在间隙里滑动 → 没有产生实际移动

    电机  ←═══  丝杠  ←═══  螺母...还在原地
                          ←─→ 间隙
                          ────────

    控制器计数器说：后退了 50 × 0.01 = 0.5mm
    实际上：螺母还在原地，0mm 都没动

    机器"以为"自己退了 0.5mm → 位置记录错了
```

```
为什么不能自动检测到：

  编码器反馈（半闭环）：
    电机轴上装旋转编码器，实时读取电机实际转了多少
    但编码器在电机端，不在螺母端
    丝杠和螺母之间的间隙，编码器测不到

    编码器  电机  丝杠  间隙  螺母
     ║       ║     ║     ↑     ║
     ║ 测得到 ║测得到║测不到║
     ║       ║     ║  这段  ║
     ║       ║     ║  不知道 ║

  直线光栅尺（全闭环）：
    在工作台上贴一把"尺子"（光栅尺）
    读数头直接测量工作台的实际位置
    这才能真正知道螺母走了多远
    但消费级机器不装这个（成本高）

    光栅尺 ══════════════════════
           ↑ 直接读实际位置
    ┌──────┼──────────────────┐
    │ 工作台│                  │
    └──────────────────────────┘
```

### 反向间隙的原理

```
丝杠和螺母之间有间隙：

  正向移动时，螺母推着丝杠的一侧：
    丝杠 →→→→ 螺母    间隙在左边

  反向移动时，螺母要先"走过"间隙，才推到另一侧：
    螺母 ←←←← 丝杠    这段空走 = 反向间隙

    正向:
    ═══════╪        ║  ← 螺母紧贴右侧
           间隙
    反向后:
    ════║        ╪════  ← 螺母紧贴左侧
              间隙
    ↑ 这段距离机器不知道，但实际位置已经偏了
```

### 补偿方法

```
虽然"不知道"，但可以"猜"：

  1. 测量：指令走 10mm → 反向走 10mm → 量一下实际差多少
     差了 0.08mm → 反向间隙 = 0.08mm

  2. 告诉机器：GRBL 设 $1=255（补偿值 0.08mm）

  3. 之后每次换方向，机器自动多走 0.08mm 来"补"

     反向前：
       计数器: 位置 = 50.00mm
     反向后：
       机器自动多发 8 个脉冲（= 0.08mm）
       控制器计数器: 位置 = 49.92mm（多扣了 0.08mm）
       但实际位置 = 50.00mm（间隙刚被补偿掉）

  这不是"检测"，而是"预设补偿"
  假设间隙固定不变（实际会磨损变化）

消费级典型值：
  3018（皮带传动）:   0.05-0.15mm
  Shapeoko（皮带）:   0.03-0.08mm
  桌面铣床（滚珠丝杠）: 0.005-0.02mm

后果：
  轮廓拐角处会有一个小偏差
  圆形可能变成略微的椭圆

解决：
  GRBL 参数 $1=255（启用反向间隙补偿）
  机器自动在换向时多走几步来弥补间隙
```

## 误差 ② — 丢步（Step Loss）

```
步进电机驱动丝杠，每个脉冲转一个小角度。
但如果切削力太大，电机转不动 → 丢掉几个脉冲 → 位置偏了。

正常：
  指令 1000 个脉冲 → 电机转 1000 步 → 精确到位

丢步：
  指令 1000 个脉冲 → 切削力太大 → 电机卡了一下 → 只转了 995 步
  → 差了 5 步 ≈ 0.025mm（16 细分下）
  → 而且这个误差会累积！越切越偏

容易丢步的情况：
  进给太快（Feed 太高）
  切深太大（ap 太大）
  加速太猛（加速度设置过高）
  刀具卡住（切屑堵住）

消费级为什么特别容易丢步：
  大部分用开环步进电机（没有位置反馈）
  机器不知道自己丢步了 → 误差累积 → 越切越偏

  工业级用伺服电机（闭环）：
  有编码器反馈 → 丢了立刻补回来 → 精度高

解决：
  降低进给和切深（保守参数）
  降低加速度（GRBL $120/$121/$122）
  不要让刀具卡住（排屑通畅）
```

## 误差 ③ — 机械变形（刚性不足）

```
切削力会让机器的框架、主轴、刀具都产生弹性变形：

  切削力 →
              ┌──┐
              │刀│  ← 刀具弯曲
             ╱│具│
           ╱  └──┘
         ╱    主轴偏移
       ╱      ↓
  ═══╤═════════════════
  ▓▓▓│▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓  ← 刀具轨迹偏了

各部分的变形量（6061铝, ap=1.5mm, ae=3mm, Fc≈63N）：

  刀具弯曲（6mm悬长30mm）: ~0.02mm
  主轴偏移:               ~0.01mm（好的主轴）
  Y轴滑块变形:            ~0.03mm（Shapeoko级）
  整体框架变形:           ~0.05mm（Shapeoko级）
  ──────────────────────────────
  总偏移:                 ~0.1mm

3018级机器（Fc ≈ 13N，但刚性更差）：
  总偏移可达 ~0.15-0.3mm

后果：
  尺寸偏大（刀被切削力推开，实际切的比设定的少）
  内角处刀偏回来 → 过切
  表面有振纹

解决：
  减小切削力（降 ap、降 Feed）
  提高机器刚性（加固、用线性导轨）
  精加工用小切深（力小 → 变形小 → 精度高）
```

## 误差 ④ — 丝杠/皮带精度

```
把旋转运动变成直线运动的机构有制造误差：

滚珠丝杠（桌面铣床）：
  精度: ±0.01mm/300mm
  每转一圈走 5mm，但实际可能是 4.998mm 或 5.002mm
  累积 300mm 后可能差 0.01mm

皮带传动（Shapeoko/3018）：
  精度: ±0.05-0.1mm/300mm
  皮带会拉伸，长时间使用后精度下降

校准方法：
  指令走 100mm → 实际量一下走了多少
  如果走了 100.3mm → 调 GRBL 参数 $100/$101/$102
  （步数/毫米，减小一点让它走准）
```

## 误差 ⑤ — 刀具本身的误差

```
刀具不是绝对精确的：

1. 直径误差
   标称直径 6mm，实际可能是 5.98mm 或 6.02mm
   → 差 0.02mm，两侧各差 0.01mm → 槽宽差 0.02mm

2. 刀具磨损
   新刀 D=6.00mm → 切 100 分钟后 D=5.95mm
   → 所有尺寸偏了 0.05mm

3. 装夹偏心（Runout）
   刀没有完全装正 → 旋转时有一点点偏摆
   → 实际切削直径比标称大

   中心
    ╷
    │ ← 偏心 0.02mm
    ○ 刀具实际旋转轨迹
   → 切出来的槽比刀径宽 0.04mm

解决：
  精加工前量一下刀具实际直径
  在 CAM 里填实际值而不是标称值
  定期换刀
```

```
│ 机械变形      │ 大           │ 中           │ 小           │
│ 传动精度      │ ±0.1mm      │ ±0.05mm      │ ±0.01mm     │
│ 刀具偏摆      │ 0.02-0.05mm │ 0.01-0.03mm │ 0.005-0.01mm│
├──────────────┼──────────────┼──────────────┼──────────────┤
│ 综合定位精度  │ ±0.1-0.3mm  │ ±0.05-0.1mm │ ±0.01-0.03mm│
│ 重复定位精度  │ ±0.05-0.1mm │ ±0.02-0.05mm│ ±0.005-0.01mm│
└──────────────┴──────────────┴──────────────┴──────────────┘

定位精度：命令走 100mm，实际走 99.9mm 还是 100.1mm
重复精度：每次走 100mm，每次的偏差是否一致

对比：
  消费级 Shapeoko: ±0.05-0.1mm → 做个外壳、支架没问题
  工业级 CNC:      ±0.005-0.01mm → 能做精密零件
  手工加工:         ±0.5-2mm → 只能做粗活
```

## 算法优化 — 按误差类型逐一对应

### 误差 ① 反向间隙 — 算法补偿

```
算法 1：反向间隙补偿（GRBL 已内置）

  1. 测量：指令前进 50mm → 反向 50mm → 量差值 = 间隙值
  2. 存入固件：GRBL $1 = 间隙值（mm）
  3. 运行时：检测到方向反转 → 自动注入额外脉冲

  伪代码：
    last_direction = NONE
    for each motion_command:
        if command.direction != last_direction:
            extra_pulses = backlash_compensation / mm_per_step
            send_pulses(extra_pulses, command.direction)
        send_pulses(command.pulses)
        last_direction = command.direction

  效果：0.1mm 间隙 → 补偿后 < 0.01mm


算法 2：丝杠螺距误差映射（Error Mapping）

  反向间隙只补偿方向切换时的固定误差。
  但丝杠在不同位置的螺距误差不同。

  1. 测量：每隔 10mm 测一次实际位置

     指令位置:  0   10   20   30   40   50   60
     实际位置:  0  10.03  20.01  29.98  40.05  50.02  59.97
     误差:      0  +0.03 +0.01 -0.02 +0.05 +0.02 -0.03

  2. 建立补偿表：
     position_map = {0: 0, 10: +0.03, 20: +0.01, ...}

  3. 运行时插值补偿：
     def compensate(target):
         lower = floor(target / 10) * 10
         upper = lower + 10
         ratio = (target - lower) / 10
         error = map[lower] * (1-ratio) + map[upper] * ratio
         return target - error

     指令 X=35 → 查表插值 → 误差 +0.015 → 实际走 X=34.985

  支持的固件：LinuxCNC（screwcomp）、grblHAL（补偿表）
  GRBL 原版不支持，需改固件或在上位机层补偿
  效果：±0.05mm → ±0.01-0.02mm
```

### 误差 ② 丢步 — 预防 + 检测

```
预防 — 在运动规划阶段避免丢步：

  算法 1：动态加速度限制

    切削时电机要同时克服惯性和切削力：
      max_accel = (motor_force - cutting_force) / mass

    切削时（有外力）→ max_accel 更小
    空跑时（无外力）→ max_accel 更大

    if is_cutting:
        accel = min(configured_accel, safe_cutting_accel)
    else:
        accel = configured_accel

  算法 2：前瞻缓冲优化（Look-ahead）

    提前看 N 段刀路，如果前面有急转弯 → 提前减速
    避免到拐角处才急刹 → 刹不住 → 丢步

    def plan_velocity(segments, lookahead=20):
        for i in range(len(segments)):
            max_entry_speed = segments[i].max_feed
            for j in range(i+1, min(i+lookahead, len(segments))):
                distance = sum(seg.length for seg in segments[i:j])
                max_speed = sqrt(segments[j].entry_speed² + 2*accel*distance)
                max_entry_speed = min(max_entry_speed, max_speed)
            segments[i].entry_speed = max_entry_speed

  算法 3：切削力估算 + 动态限速

    加工前估算每个刀路段的切削力
    力大的段落自动降低 Feed

    for each segment:
        force = Kc * ap * ae * fz
        if force > safe_force:
            segment.feed *= safe_force / force

检测 — 丢了之后的补救：

  算法 4：位置验证检查点

    在 G-code 关键位置插入回零验证：
      加工一个特征 → 回参考点 → 探针验证位置
      偏了就补偿，偏太多就停机报警

  算法 5：电流/振动监测

    电机驱动芯片采样电流 → 持续超阈值 → 可能丢步 → 自动减速
    机架上装 IMU 加速度传感器 → 振动模式突变 → 异常检测
    （消费级尚未普及，研究阶段）
```

### 误差 ③ 机械变形 — 力建模补偿

```
算法 1：刀具偏转预补偿

  切削力把刀推开 → 切的比设定的少
  提前算出偏移量 → 反方向预偏 → 抵消误差

  1. 计算切削力：Fc = Kc × ap × ae × fz
  2. 计算偏转量：
     δ = F × L³ / (3 × E × I)
     I = π × D⁴ / 64（刀具截面惯性矩）
  3. 实测法更可靠：
     切一个标准件 → 量实际尺寸 → 算出偏移量
  4. 在 CAM 里把所有轮廓尺寸缩小偏移量
     刀被推开的部分恰好抵消预偏 → 实际尺寸精准

  适合批量生产（固定参数下一次性标定）
  精加工才需要（粗加工不在乎 0.1mm 偏差）


算法 2：拐角自适应减速

  拐角处 ae 突增 → 切削力翻倍 → 变形增加

  for each segment:
      force = estimate_force(segment.ap, segment.ae, segment.fz)
      if force > force_threshold:
          reduction = force_threshold / force
          segment.feed *= reduction

  拐角处 Feed 从 960 降到 480
  → Fc 减半 → 变形减半 → 精度提高
  总加工时间增加 ~10%（拐角只占少量时间）

  CAM 支持：Fusion 360 "Feed Rate Optimization"
            Vectric "Corner Feeds"


算法 3：粗加工 + 精加工分离

  粗加工：大切深、快进给 → 力大 → 变形大 → 但留余量
  精加工：小切深、慢进给 → 力小 → 变形小 → 切到最终尺寸

  粗加工留下 0.1-0.3mm 余量（包含变形误差）
  精加工只切这 0.1-0.3mm → 力极小 → 精度最高
```

### 误差 ④ 丝杠/皮带精度 — 校准 + 映射

```
算法 1：线性校准（GRBL 已内置）

  指令走 100mm → 实际量一下走了多少
  偏了 → 调 GRBL $100/$101/$102（步数/毫米）

  例：指令 100mm，实际走了 100.3mm
  steps_per_mm = 原值 × 100 / 100.3
  → 全行程线性误差消除

  限制：只补偿平均误差，不补偿位置相关的误差


算法 2：丝杠误差映射（见误差 ① 的算法 2）

  分段测量 → 建立补偿表 → 运行时逐点补偿
  消除位置相关的非线性误差
```

### 误差 ⑤ 刀具误差 — 直径/磨损/偏心

```
算法 1：实际直径校准

  新刀标称 6mm → 用千分尺量实际值（如 5.98mm）
  CAM 里填 5.98mm 而不是 6.00mm
  → 刀路自动按实际直径计算 → 尺寸准确


算法 2：刀具磨损跟踪

  1. 记录累计切削量：total_volume = Σ(ap × ae × distance)
  2. 磨损模型：wear ≈ k × total_volume / D
     铝合金: k ≈ 0.0001
     钢:     k ≈ 0.0005
  3. 预测当前有效直径：actual_D = nominal_D - wear
  4. 定期实测校准（每加工 N 个零件后量一下）

  消费级实际做法：定期量刀具 → 手动更新 CAM 刀径值


算法 3：偏心（Runout）补偿

  刀具装进夹头后旋转中心不一定和几何中心重合：
    标称 D=6mm，偏心 0.02mm
    → 实际切削直径 = 6.0 + 2×0.02 = 6.04mm

  测量方法：
    千分表法：测针对着刀刃，主轴慢转一圈 → 偏心量=(最大-最小)/2
    试切法：切一条直线 → 量实际槽宽 → 偏心量=(槽宽-刀径)/2

  补偿：
    effective_D = nominal_D + 2 * runout
    CAM 里填 effective_D → 补偿完成

  进阶：偏心有角度依赖性（某角度偏得多某角度少）
  完整补偿需要主轴编码器（消费级没有）
  → 简化处理：用平均偏心量补偿，够用
```

## 怎么提高精度（硬件层）

```
1. 校准（零成本）
   测量实际移动距离 → 调 GRBL 步数/毫米参数
   反向间隙补偿 → 调 GRBL $1 参数
   消除最大的系统性误差

2. 保守参数（零成本）
   精加工时减小 ap 和 Feed → 降低切削力 → 减少变形
   粗加工追求效率，精加工追求精度

3. 两次加工（零成本）
   粗加工留 0.2mm 余量
   精加工小切深切除余量 → 力小 → 变形小 → 精度高

4. 升级硬件（有成本）
   皮带 → 滚珠丝杠：消除反向间隙
   V轮 → 线性导轨：提高刚性
   步进电机 → 伺服电机：消除丢步

5. 维护（低成本）
   定期检查皮带张力
   定期润滑丝杠
   定期检查刀具磨损
```
