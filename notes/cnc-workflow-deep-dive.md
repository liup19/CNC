# CNC 加工全流程详解：CAD → CAM → G-code → 控制器 → 加工

> 最后更新：2025-05-15

---

## 1. CAD（Computer-Aided Design）— 设计

**做什么：** 定义"你要做什么形状"。

用户用 CAD 软件创建工件的 2D/3D 几何模型。输出的不是"怎么加工"，而是"最终长什么样"。

### 关键文件格式

| 格式 | 类型 | 说明 |
|------|------|------|
| SVG / DXF | 2D 矢量 | 平面雕刻、轮廓切割，最简单 |
| STL | 3D 网格 | 只有表面三角面片，无拓扑信息 |
| STEP / IGES | 3D 实体 | 工业标准，保留精确曲面和拓扑 |

### 消费级常见工具

| 工具 | 价格 | 2D/3D | 特点 |
|------|------|-------|------|
| Easel | 免费 | 2D/2.5D | 最简单，网页端，适合入门 |
| Carbide Create | 免费 | 2D/2.5D | Shapeoko 配套，离线可用 |
| Fusion 360 | 免费(个人版) | 3D | 功能强大但学习曲线陡 |
| FreeCAD | 免费开源 | 3D | 参数化建模，界面较粗糙 |
| Inkscape / Illustrator | 免费/付费 | 2D | 矢量绘图，导出 SVG/DXF |

### 消费级痛点

- 2D/2.5D 场景（平面雕刻、轮廓切割）相对简单，SVG/DXF 就够了
- 3D 浮雕、曲面加工才需要 Fusion 360 / FreeCAD，学习门槛陡增
- 很多用户只想"照着一张图刻出来"，但被迫学 CAD
- **潜在简化：** 提供图库/模板 + 简单编辑器，跳过 CAD 或降低 CAD 门槛

## 2. CAM（Computer-Aided Manufacturing）— 制造规划

**做什么：** 把"要做什么形状"转化为"刀具怎么走"。

CAM 是全链路中**技术门槛最高、对消费者最不友好**的环节。用户需要理解大量切削工艺知识。

### 2.1 刀具选择

| 刀具类型 | 用途 | 特点 |
|----------|------|------|
| 平底铣刀 (Flat End Mill) | 轮廓切割、平面铣削 | 最通用 |
| 球头铣刀 (Ball Nose) | 3D 曲面、浮雕 | 留下弧形底 |
| V 型刀 (V-Bit) | 倒角、文字雕刻 | 角度 60°/90° 常见 |
| 雕刻刀 (Engraving Bit) | 细线雕刻 | 刀尖极细 |

### 2.2 核心切削参数

这是让新手最困惑的部分 — 需要根据材料和刀具计算一堆参数：

| 参数 | 含义 | 典型值（木材） | 典型值（铝） |
|------|------|---------------|-------------|
| **主轴转速 (RPM)** | 刀具每分钟转多少圈 | 10,000–20,000 | 5,000–15,000 |
| **进给率 (Feed Rate)** | 刀具每分钟移动多快 | 500–1,500 mm/min | 200–800 mm/min |
| **切深 (Depth of Cut)** | 每刀吃多深 | 1–5 mm | 0.1–1 mm |
| **步距 (Stepover)** | 相邻刀路重叠多少 | 刀具直径的 30–50% | 刀具直径的 10–40% |
| **切宽 (Width of Cut)** | 每刀横向切多宽 | 取决于特征 | 取决于特征 |

这些参数之间的关系（简化版）：

```
切削力 ∝ 进给率 × 切深 × 切宽 / 转速

切屑负载 (Chip Load) = 进给率 / (转速 × 刀刃数)
                    ↑ 这个值太大 → 断刀
                    ↑ 这个值太小 → 刀具摩擦发热，烧坏
```

**一个真实的例子：** 用户想用 6mm 平底刀切铝合金，需要知道转速设 10000、进给设 400、切深不超过 0.5mm… 普通消费者根本不知道这些。

### 2.3 刀具路径策略

CAM 软件根据几何形状和参数，生成刀具的运动轨迹：

| 策略 | 用途 | 图形化理解 |
|------|------|-----------|
| 轮廓切割 (Profile) | 沿外形切出零件 | 刀沿边缘走一圈 |
| 挖槽 (Pocket) | 铣出一个凹槽 | 一层一层往里挖 |
| 平面铣 (Facing) | 把表面铣平 | 来回扫描式走刀 |
| 钻孔 (Drilling) | 打孔 | 下刀 → 抬刀 → 换位置 |
| 3D 平行铣 (Parallel) | 3D 曲面粗加工 | 来回走平行线 |
| 3D 等高 (Contour) | 3D 曲面精加工 | 沿等高线一圈圈走 |
| 自适应清理 (Adaptive) | 高效粗加工 | 摆线式走刀，切削力恒定 |

### 2.4 消费级 CAM 工具

| 工具 | 价格 | 能力 | 痛点 |
|------|------|------|------|
| Easel | 免费 | 2D/2.5D | 功能有限，复杂刀路做不了 |
| Carbide Create | 免费 | 2D/2.5D | 工作流不够直观 |
| Fusion 360 CAM | 免费(个人版) | 全功能 | 学习曲线极陡，需理解所有参数 |
| Carveco Maker | $15/月 | 2D/2.5D | 中间路线 |
| Vectric VCarve | $349 买断 | 2D/2.5D | 好用但要花钱 |
| FreeCAD Path | 免费开源 | 2D/3D | 不够成熟，bug 多 |

**核心痛点：CAM 是消费级 CNC 的最大障碍。** 用户需要同时懂材料、刀具、切削力学，才能正确设置参数。

## 3. G-code — 机器指令

**做什么：** 把 CAM 生成的刀具路径翻译成机器能理解的逐行指令。

G-code 是一种古老的文本格式（1950 年代发明），每一行告诉机器做一个原子动作。

### 3.1 G-code 结构示例

```gcode
; === 头部：初始化 ===
G21              ; 设置单位为毫米
G90              ; 绝对坐标模式
G17              ; 选择 XY 平面

; === 换刀和设置 ===
T1               ; 选择 1 号刀具
M6               ; 执行换刀
S12000 M3        ; 主轴转速 12000 RPM，顺时针启动
G4 P2            ; 暂停 2 秒（等主轴达到转速）

; === 移动到安全高度 ===
G0 Z5            ; 快速移动到 Z=5mm（安全高度）

; === 第一刀：移动到起点 ===
G0 X10 Y10       ; 快速移动到 X=10, Y=10
G1 Z-1 F100      ; 以 100mm/min 进给速度下刀到 Z=-1

; === 轮廓切割 ===
G1 X50 Y10 F600  ; 以 600mm/min 直线进给到 (50, 10)
G1 X50 Y50       ; 直线到 (50, 50)
G2 X10 Y50 I-20 J0  ; 顺时针圆弧到 (10, 50)
G1 X10 Y10       ; 直线回到起点

; === 抬刀 ===
G0 Z5            ; 快速抬刀到安全高度

; === 收尾 ===
M5               ; 主轴停止
G0 X0 Y0         ; 回原点
M2               ; 程序结束
```

### 3.2 常用 G-code 速查

| 代码 | 含义 | 类型 |
|------|------|------|
| G0 | 快速定位（不切削，空跑） | 运动指令 |
| G1 | 直线进给（切削移动） | 运动指令 |
| G2 | 顺时针圆弧进给 | 运动指令 |
| G3 | 逆时针圆弧进给 | 运动指令 |
| G17/18/19 | 选择 XY/ZX/YZ 平面 | 平面选择 |
| G20 | 英寸单位 | 单位 |
| G21 | 毫米单位 | 单位 |
| G28 | 回参考点（Home） | 位置 |
| G90 | 绝对坐标 | 坐标模式 |
| G91 | 相对坐标 | 坐标模式 |
| M3 | 主轴正转 | 辅助指令 |
| M5 | 主轴停止 | 辅助指令 |
| M6 | 换刀 | 辅助指令 |
| M8/M9 | 冷却液开/关 | 辅助指令 |

### 3.3 后处理（Post-processing）

CAM 输出的不是直接可用的 G-code，需要经过**后处理器**转换：

```
CAM 输出（通用刀路数据）
        ↓
    后处理器（Post-processor）
        ↓
特定机器的 G-code（考虑固件差异）
```

不同固件/控制器的 G-code 方言不同：

| 固件/控制器 | G-code 方言差异 |
|-------------|----------------|
| GRBL | 精简子集，不支持某些高级指令 |
| Mach3 | 支持较完整，自定义 M code |
| LinuxCNC | 最完整，支持高级功能 |
| Fanuc/Haas | 工业标准，消费级不涉及 |

**消费级痛点：** 用户经常在后处理环节出错 — 选错了后处理器，G-code 格式不匹配，机器行为异常。

## 4. 控制器（Controller）— 实时执行

**做什么：** 逐行读取 G-code，实时计算电机运动，发送脉冲信号驱动电机。

这是整个链路中**最硬核的技术环节**，涉及实时运动控制算法。

### 4.1 控制器的工作原理

```
G-code 文件
    ↓ 逐行解析
指令解释器 (Parser)
    ↓ 提取运动参数
    ├── 目标位置 (X, Y, Z)
    ├── 进给速度 (F)
    └── 主轴转速 (S)
    ↓
运动规划器 (Motion Planner)
    ↓ 计算轨迹
    ├── 加减速曲线（S-curve / Trapezoidal）
    ├── 前瞻缓冲（Look-ahead，提前看后续指令）
    └── 速度衔接（段间速度平滑过渡）
    ↓
插补器 (Interpolator)
    ↓ 实时计算各轴位置
    ├── 直线插补（G1）：按比例分配各轴脉冲
    ├── 圆弧插补（G2/G3）：实时计算弧上各点
    └── 通常 1ms 一个插补周期
    ↓
脉冲发生器 (Step Generator)
    ↓ 发出步进/伺服脉冲
    ├── 步进电机：方向(Dir) + 脉冲(Step) 信号
    └── 伺服电机：PWM / 闭环控制信号
    ↓
电机驱动器 → 电机 → 机械运动
```

### 4.2 关键技术概念

#### 前瞻缓冲（Look-ahead）

控制器不只看当前一行 G-code，而是提前读取数百行：

- 如果当前段末速度很高、下一段要急转弯 → 提前减速
- 如果后续都是直线 → 可以保持高速
- **缓冲区越大，运动越平滑**，但需要更多内存

```
速度
  │    ╱╲      ╱──────╲
  │   ╱  ╲    ╱        ╲
  │  ╱    ╲  ╱          ╲
  │ ╱      ╲╱            ╲
  └────────────────────────→ 时间
  没有前瞻: 锯齿状加减速（抖动）

  有前瞻: 平滑的 S 曲线（流畅）
```

#### 加减速控制

| 方式 | 特点 | 适用场景 |
|------|------|----------|
| 梯形加减速 | 简单，计算量小 | 入门级控制器 |
| S 曲线加减速 | 平滑，无突变，精度高 | 中高端控制器 |
| 速度前瞻 + S 曲线 | 最优运动性能 | 高端消费级 |

#### 脉冲频率

- 步进电机每转的步数 × 目标转速 = 需要的脉冲频率
- 例：200 步/转 × 16 细分 × 10 转/秒 = 32,000 Hz
- GRBL 在 Arduino Uno (16MHz) 上能产生 ~30kHz 稳定脉冲
- ESP32 (240MHz 双核) 可以做到 100kHz+

### 4.3 消费级控制器选项

| 方案 | 硬件 | 脉冲频率 | 特点 |
|------|------|----------|------|
| GRBL | Arduino Uno (ATmega328) | ~30kHz | 最成熟，社区最大 |
| GRBL on Arduino Mega | ATmega2560 | ~30kHz | 更多 I/O |
| FluidNC | ESP32 | ~100kHz | WiFi、多轴、更灵活 |
| grblHAL | 各种 ARM Cortex-M | 100kHz+ | 高性能，社区活跃 |
| Mach3 | PC + 并口 | N/A | 老牌软件，逐步淘汰 |
| LinuxCNC | PC + 专用 I/O 卡 | N/A | 工业级，消费级过重 |

### 4.4 G-code 发送软件

控制器固件只负责"执行"，需要一个上位机软件把 G-code 文件发送给控制器：

| 软件 | 平台 | 特点 |
|------|------|------|
| UGS (Universal Gcode Sender) | Java 跨平台 | 最流行的 GRBL 发送器 |
| bCNC | Python | 功能丰富，有探针功能 |
| Candle | Windows | 简洁，GRBL 专用 |
| Easel | Web | 一体化，集成 CAD/CAM/发送 |
| Carbide Motion | 桌面 | Shapeoko 专用 |

**消费级痛点：** 用户需要在 CAM 软件导出 G-code → 打开发送器软件 → 连接机器 → 手动设零点 → 发送加工。流程碎片化。

## 5. 加工（Machining）— 物理切削

**做什么：** 刀具实际切削材料，把工件加工成设计形状。

### 5.1 加工前的准备

```
1. 装夹工件
   └── 把原材料固定在工作台上（夹具/双面胶/压板）

2. 安装刀具
   └── 把正确的刀具装入主轴夹头（ER11 等）

3. 设零点（Zeroing / Tool Setting）
   ├── 手动：慢慢摇到材料表面，设为 Z=0
   └── 自动：Z-probe 触碰式自动检测

4. 设工作原点（Work Offset）
   └── G54-G59 指令，定义工件坐标系原点

5. 预跑检查（Air Cut）
   └── 抬高 Z 轴空跑一遍，确认刀路不会撞到夹具等
```

### 5.2 加工过程中的物理现象

#### 切削力

```
切削力 (N) = 材料比切削力 × 切屑截面积
           = Ks × (切深 × 切宽 × 每齿进给)
```

- 铝合金：切削力较小，消费级机器可以应对
- 钢铁：切削力大，消费级机器刚性不够
- 木材/MDF：切削力很小，但粉尘大

#### 切削热

- 刀具-材料摩擦产生热量
- 铝合金切削热较高，需要冷却或低进给
- 木材不怕热，但高温会烧焦

#### 振动和颤振（Chatter）

- 切削力周期性变化 → 引起机床振动
- 当振动频率接近机床固有频率 → 颤振（共振）
- 表现：加工表面出现波纹状纹路，噪音尖锐
- 解决：降低切深、降低进给、改变转速（避开共振频率）

### 5.3 材料特性（消费级常见）

| 材料 | 可加工性 | 切削参数特点 | 注意事项 |
|------|----------|-------------|----------|
| 木材/MDF | 容易 | 高转速、高进给、大切深 | 粉尘大，需除尘 |
| 亚克力 (PMMA) | 中等 | 中转速、低进给 | 容易融化粘连，需降温 |
| 铝合金 (6061) | 中等 | 中转速、低进给、小切深 | 切屑易粘刀，需润滑 |
| 黄铜 | 容易 | 高转速、中进给 | 自润滑，好加工 |
| 铜 | 中等 | 低转速、低进给 | 软而粘，容易缠屑 |
| 钢铁 | 困难 | 低转速、极低进给 | 消费级机器通常刚性不够 |

### 5.4 加工策略：粗加工 + 精加工

实际加工通常分两步：

```
粗加工（Roughing）：
├── 目标：快速去除大量材料
├── 大切深、大步距、快速进给
├── 表面粗糙，留 0.1-0.5mm 余量
└── 刀具：平底铣刀

精加工（Finishing）：
├── 目标：达到最终精度和表面质量
├── 小切深、小步距、慢进给
├── 达到设计尺寸和表面光洁度
└── 刀具：球头铣刀 / 平底铣刀
```

### 5.5 消费级加工常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 断刀 | 切屑负载过大 / 刀具卡住 | 降低进给或切深 |
| 表面粗糙 | 颤振 / 步距过大 | 减小步距，检查装夹刚性 |
| 尺寸不准 | 机器刚性不足 / 丝杠间隙 | 校准丝杠反向间隙补偿 |
| 材料烧焦 (木材) | 进给太慢 / 转速太高 | 降低转速或提高进给 |
| 刀具磨损快 | 切削参数不当 / 无冷却 | 调整参数，加冷却液 |
| 工件松动 | 装夹不牢 / 切削力太大 | 改善装夹，减小切削力 |

## 总结：当前现实 vs 理想体验

### 当前现实（5+ 个独立工具）

```
用户脑中的想法
    ↓
[1. CAD 软件] → 设计 2D/3D 模型 → 导出 SVG/STL/STEP
    ↓
[2. CAM 软件] → 选刀具、设参数、生成刀路 → 导出 G-code
    ↓ 切换软件
[3. G-code 发送器] → 连接机器、设零点、发送加工
    ↓
[4. 控制器固件] → 实时执行
    ↓
[5. 物理加工] → 切削、冷却、除尘
    ↓
成品（可能需要多次返工调整参数）
```

### 理想体验（一体化，类似 3D 打印机）

```
用户脑中的想法
    ↓
[一体化软件]
    ├── 导入设计（SVG/STL/图片）
    ├── AI 自动推荐刀具和参数
    ├── 3D 仿真预览结果
    └── 一键加工
    ↓
[智能硬件]
    ├── 自动识别刀具（RFID）
    ├── 自动对刀（Z-probe）
    ├── 实时监测（力传感器异常停机）
    └── 自动除尘
    ↓
成品
```

### 差距在哪

- 3D 打印机只需要一个切片软件（Cura/PrusaSlicer），用户选材料 → 选质量 → 打印
- CNC 需要用户同时理解材料力学、刀具几何、切削动力学 — 这是根本性的复杂度差异
- **AI 可以帮助弥合这个差距**：把专业切削知识编码成智能推荐系统
