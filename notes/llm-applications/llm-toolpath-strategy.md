# LLM 实现刀具路径策略选择

> 最后更新：2026-05-16

---

## 核心难题

LLM **看不懂** CAD 文件（STEP/STL/SVG 是二进制或几何数据）。所以不能把文件直接丢给 LLM 让它"看"。

解决思路：**先用确定性代码提取几何特征 → 转成文字/结构化描述 → LLM 做推理决策。**

```
总体架构：

  CAD 文件 (SVG/DXF/STL/STEP)
      ↓
  ┌─────────────────────┐
  │ 几何特征提取器       │  ← 确定性代码，不用 LLM
  │ (Python 库做计算)    │
  └─────────┬───────────┘
            ↓
  结构化特征描述（JSON）
      ↓
  ┌─────────────────────┐
  │ LLM 推理层          │  ← 这里用大模型
  │ 策略 + 刀具 + 顺序   │
  └─────────┬───────────┘
            ↓
  加工方案（JSON）
      ↓
  ┌─────────────────────┐
  │ 确定性验证           │  ← 安全检查
  └─────────────────────┘
            ↓
  最终输出
```

---

## 第一步：几何特征提取（确定性代码）

### 2D 文件（SVG/DXF）— 最简单，消费级最常见

用 `ezdxf` / `svgpathtools` 库解析。

```json
{
  "file": "bracket.svg",
  "bounding_box": [80, 60],
  "features": [
    {
      "type": "closed_profile",
      "shape": "rectangle_with_fillet",
      "size": [80, 60],
      "corner_radius": 3,
      "depth": 5
    },
    {
      "type": "pocket",
      "shape": "rectangle",
      "size": [30, 20],
      "position": [25, 20],
      "depth": 3,
      "corner_radius": 2
    },
    {
      "type": "circle",
      "diameter": 4,
      "position": [10, 10],
      "through": true
    },
    {
      "type": "circle",
      "diameter": 4,
      "position": [70, 10],
      "through": true
    }
  ]
}
```

特征识别规则（确定性代码）：

```
封闭曲线在内部     → pocket（挖槽）
封闭曲线是最外圈   → profile（轮廓切割）
小圆（D < 10mm）   → drill（钻孔）
文字/线条          → engrave（雕刻）
带深度标注的       → 记录目标深度
```

### 3D 文件（STL/STEP）— 更复杂

用 `trimesh` / `cadquery` / `opencascade` 库。

提取内容：

- 整体包围盒尺寸
- 表面特征（平面/曲面/斜面）
- 拓扑特征（通过几何分析识别 pocket/boss/hole）
- 最小内角半径（决定最小刀具直径）
- 最大加工深度

## 第二步：LLM 推理（策略选择的核心）

### 给 LLM 的输入

1. 特征描述（上面提取的 JSON）
2. 机器能力（刚性等级、RPM 范围、轴数）
3. 可用刀具列表（用户有什么刀）
4. 材料（决定参数范围）
5. 知识库上下文（RAG 检索的策略规则）

### Prompt 示例

```
SYSTEM:
你是 CNC 加工工艺规划师。
根据零件特征描述、机器能力和可用刀具，制定加工方案。

规则：
- 每个特征必须分配一种策略
- 粗加工在精加工之前
- 内角半径决定最小刀具直径
- 刀具直径应尽量大（效率高）
- 刚性低的机器要保守

USER:
材料: 6061 铝合金
机器: Shapeoko 4 (MEDIUM 刚性, 3轴, 12000 RPM)
可用刀具:
  - 6mm 2刃硬质合金平底刀
  - 3mm 2刃硬质合金平底刀
  - 60° V型刀

零件特征:
1. 外轮廓: 80×60mm 矩形, 圆角 R3, 深 5mm (穿透)
2. 内部挖槽: 30×20mm 矩形, 深 3mm, 圆角 R2
3. 孔 ×2: 直径 4mm, 穿透

请制定加工方案。
```

### LLM 输出示例

```json
{
  "operations": [
    {
      "step": 1,
      "strategy": "facing",
      "feature": "顶部表面",
      "tool": "6mm_flat",
      "reason": "先铣平顶面，保证基准面平整"
    },
    {
      "step": 2,
      "strategy": "pocket",
      "feature": "feature_2 (30×20 挖槽)",
      "tool": "3mm_flat",
      "reason": "内角 R2mm，6mm刀切不过去，必须用 3mm刀",
      "notes": "分3层，每层 1mm"
    },
    {
      "step": 3,
      "strategy": "drill",
      "feature": "feature_3,4 (2×Ø4孔)",
      "tool": "3mm_flat",
      "reason": "没有钻头，用 3mm平底刀螺旋下刀代替钻孔",
      "notes": "螺旋下刀，每圈 0.5mm"
    },
    {
      "step": 4,
      "strategy": "profile",
      "feature": "feature_1 (外轮廓)",
      "tool": "6mm_flat",
      "reason": "外轮廓圆角 R3mm，6mm刀可以切（R3 > D/2=3 ✓ 刚好）",
      "notes": "最后切，切完零件就脱落。需要加 tab 防止飞出",
      "warnings": ["必须添加连接片(tab)，否则零件切断后飞出"]
    }
  ],
  "total_estimated_time": "约 15 分钟",
  "setup_notes": "需要双面胶或压板固定，最后切轮廓时注意 tab"
}
```

---

## LLM 在这里具体做了什么推理

### 1. 内角半径 → 刀具匹配

```
内角 R2mm → 6mm刀半径 3mm → 放不进去 → 必须用 3mm刀
外角 R3mm → 6mm刀半径 3mm → 刚好能切 → 可以用 6mm刀

LLM 擅长：比较数值 + 应用规则
```

### 2. 操作顺序推理

```
先铣面 → 再挖内部 → 最后切外轮廓

原因：切外轮廓后零件会松动，其他操作必须在零件固定时完成

LLM 擅长：因果推理
```

### 3. 缺少工具的替代方案

```
没有 4mm钻头，但有 3mm平底刀
→ 用 3mm刀螺旋下刀代替钻孔

LLM 擅长：基于约束的创新解决
```

### 4. 安全提示

```
最后切轮廓 → 零件会飞出 → 需要 tab

LLM 擅长：风险识别 + 经验知识
```

## 第三步：确定性验证（安全网）

LLM 输出必须经过验证：

```
✓ 刀具直径 ≤ 内角半径 × 2（否则切不过去）
✓ 刀具在可用列表中（LLM 不能"发明"刀具）
✓ 切深不超过 ap_max
✓ 切削力不超过机器极限
✓ 操作顺序合理（粗加工在精加工前）

验证失败 → 反馈给 LLM → 要求修正
```

---

## LLM 不可替代的部分 vs LLM 擅长的部分

```
确定性代码做（LLM 做不了）：
  ├── 几何特征提取（需要数学计算）
  ├── 刀具路径生成（需要坐标计算）
  ├── 碰撞检测（需要几何推理）
  └── 参数精确计算（RPM/Feed/Force）

LLM 做（确定性代码做不好）：
  ├── 把特征映射到策略（需要经验知识）
  ├── 处理约束冲突（没有钻头怎么办）
  ├── 操作排序（需要因果推理）
  ├── 风险识别（需要经验）
  └── 自然语言解释（为什么选这个方案）

用户交互（LLM 独有价值）：
  用户："我只有一把 6mm刀，能做吗？"
  LLM："可以，但内角 R2 的槽做不了（6mm刀放不进去），
        可以改成 R3 的圆角，或者用这把刀铣个方角。
        要不要帮你改设计？"
```

## 消费级 2.5D 的简化版

消费级大部分是 2.5D（平面图形 + 不同深度），策略很有限：

```
2.5D 的策略只有这几种：

  封闭图形 + 有深度 = pocket（挖槽）
  封闭图形 + 穿透   = profile（轮廓切割）
  圆形 + 小直径     = drill（钻孔）
  文字/线条 + 浅深度 = engrave（雕刻）
  整个表面          = facing（平面铣）
```

策略选择可以用规则引擎覆盖 90%：

```python
if feature.type == "circle" and feature.diameter < 10:
    strategy = "drill"
elif feature.type == "closed_profile" and feature.is_outermost:
    strategy = "profile"
elif feature.type == "closed_profile" and not feature.is_outermost:
    strategy = "pocket"
```

LLM 只在以下情况介入：

- 规则引擎无法判断（模糊情况）
- 需要解释和交互
- 约束冲突需要取舍
- 用户提出特殊需求

## 没有真实 CAD 模型的问题

上面的 Prompt 示例直接给了文字描述的特征，LLM 就输出了方案。但这是**不完整**的流程 —— 缺少了 CAD 文件和确定性特征提取。

### 为什么不能跳过 CAD 直接用文字描述

```
问题 1 — 特征描述不准确
  人说"圆角 R3"，实际 CAD 里可能是 R2.5
  → 6mm刀以为能切过去，实际卡住

问题 2 — 遗漏特征
  人描述了 3 个特征，但 CAD 里可能有第 4 个：
    比如有个 M3 螺纹孔，描述时忘了写
    → 没给这个孔分配策略 → 漏加工

问题 3 — 特征冲突检测不到
  挖槽 30×20mm，深 3mm
  但如果材料总厚度只有 3.5mm
  → 槽底只剩 0.5mm → 极薄，切削力会把工件顶穿
  → 文字描述里没说材料厚度

问题 4 — 尺寸冲突
  孔位 (10,10) 离边缘只有 10mm
  6mm刀切外轮廓时，刀心到边缘 3mm
  → 孔壁和轮廓之间只剩 10 - 3 - 2 = 5mm
  → 如果描述里的位置有误 → 孔和轮廓铣穿

问题 5 — 刀具干涉
  没有模型就做不了碰撞检测
  → 刀具夹头会不会撞到工件？不知道
  → 刀具移动时会不会碰到已加工完的凸台？不知道
```

### 正确流程：必须经过 CAD 文件 + 确定性提取

```
用户有 CAD 文件（SVG/DXF）的情况：

  用户上传 bracket.svg
      ↓
  ┌─────────────────────────────────────┐
  │ 确定性代码分析 SVG                   │
  │                                     │
  │  解析所有路径:                       │
  │    - 闭合矩形 (80.0 × 60.0mm)       │  ← 精确数值，不是人写的
  │    - 4 个圆弧 (R = 3.0mm)           │  ← 精确数值
  │    - 闭合矩形 (30.0 × 20.0mm)       │
  │    - 4 个圆弧 (R = 2.0mm)           │
  │    - 圆 (D = 4.0mm) × 2             │
  │                                     │
  │  计算空间关系:                       │
  │    - 最小壁厚: 7mm (槽到边缘)        │  ← 自动发现
  │    - 最小间距: 15mm (孔到边缘)       │  ← 自动发现
  │    - 内角最小半径: R2                │  ← 自动发现
  │                                     │
  │  验证:                               │
  │    - 所有闭合路径检查完整性          │
  │    - 特征之间无重叠/冲突             │
  └──────────────┬──────────────────────┘
                 ↓
  展示给用户确认：
    "检测到以下特征：
     1. 外轮廓矩形 80×60mm, 4×R3圆角
     2. 内部矩形 30×20mm, 4×R2圆角
     3. 圆孔 Ø4mm × 2
     最小壁厚: 7mm
     以上是否正确？"
                 ↓
  用户确认 ✓
                 ↓
  LLM 基于确认后的数据制定方案


用户没有 CAD 文件的情况（纯对话）：

  用户："我想做一个手机支架，大概 80×60mm，
        中间有个槽放手机，两边各有螺丝孔"

  → 这时候不能直接出加工方案
  → 应该先帮用户生成 CAD：

  LLM："我帮你画，确认一下：
        1. 槽的尺寸？手机多宽？
        2. 孔要多大？M3 还是 M4？
        3. 材料厚度？"

  → 收集需求 → 生成 CAD 脚本（CadQuery/OpenSCAD）
  → 渲染成 SVG/STL
  → 然后走上面的流程
```

### 类比

```
没有模型就出方案，等于：

  医生没看 CT 片，听病人描述症状就开了药
  病人说"我肚子右边疼"→ 医生开了阑尾炎的药
  但病人可能只是吃坏了肚子

  正确流程：
  病人描述 → 医生看 CT → 确认诊断 → 开药
  用户描述 → 代码分析 CAD → 确认特征 → 出方案

  没有模型就出方案，等于跳过诊断直接开药
```

---

## 总结

LLM 不直接分析几何，而是接收结构化的特征描述做高层决策。几何分析用确定性代码，策略推理和自然语言交互用 LLM。两者配合：确定性代码负责"算"，LLM 负责"想"和"说"。

**关键前提：特征数据必须来自确定性代码对 CAD 文件的分析，不能依赖人的文字描述。没有模型就不出方案，先造模型再分析。**
